Lab 07: Flask & PostgreSQL

In the Flaskr app, you leveraged Python and Flask's built-in support for SQLite. But we spent the first half of the year learning about PostgreSQL, so let's figure out a way to connect it to our flask-todo project.

You're going to use a tool called an adapter, a library written in your coding language that allows you to send SQL queries a running DBMS. In this case, you'll use psycopg.

Requirements

By the end of class, you should have a functioning web app that does the following:

  • Shows a list of to-do items that have been stored in a database. Each item should have the following data:
    • It's text content
    • When it was created
    • If it's complete or not
  • Uses a form to add new items to the above list. It should only ask for the text content of the new item.
  • Can update existing items mark them as completed.

Things to Think About

  • Use the example flaskr tutorial app to see where the database functionality will need to go. The code will not be the same since we aren't using SQLite, but the concepts still apply.
  • If it is hard to think about the three pieces of this app all at once (the flask portion, the database portion, and the items themselves), it might be a good idea to break things out into modules. You can even use OOP principles to package up parts of your app.
  • If you have to do a lot of manual setup for your database connection, remember that your fellow developers will have to follow those instructions. A better idea is to rely on code. Look at how the flaskr tutorial app handles the database setup using code and simple commands, rather than a long list of instructions to follow in pgAdmin.
